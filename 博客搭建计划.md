# Hexo 个人博客搭建计划

> 目标：将当前 Hexo 项目部署到 GitHub Pages，通过 `https://dhwass01.github.io` 访问

---

## 一、前期准备

### 1.1 确认环境

- [x] Node.js 已安装（项目中已有 `node_modules`）
- [x] Hexo v7.3.0 已安装（本地）
- [x] `hexo-deployer-git` 已安装
- [ ] Git 已安装并配置用户名/邮箱

### 1.2 确认 GitHub 仓库

- [ ] 登录 GitHub，确认仓库 `Dhwass01/Dhwass01.github.io` 已存在
- [ ] 仓库必须命名为 `<用户名>.github.io`，这样 GitHub Pages 才会自动识别

---

## 二、配置 Hexo

### 2.1 修改站点配置 `_config.yml`

```yaml
# 修改 URL 为你的 GitHub Pages 地址
url: https://dhwass01.github.io
root: /

# 部署配置
deploy:
  type: git
  repo: https://github.com/Dhwass01/Dhwass01.github.io.git
  branch: main
```

### 2.2 可选：完善站点信息

```yaml
title: 你的博客标题
subtitle: 博客副标题
description: 博客描述
author: Dhwass01
language: zh-CN
```

---

## 三、初始化 Git 并关联仓库

### 3.1 在项目目录初始化 Git

```bash
cd d:\BlogFile
git init
```

### 3.2 关联远程仓库

```bash
git remote add origin https://github.com/Dhwass01/Dhwass01.github.io.git
```

### 3.3 创建 `.gitignore` 文件

在项目根目录创建 `.gitignore`，内容如下：

```
.DS_Store
Thumbs.db
db.json
*.log
node_modules/
.deploy*/
public/
```

> 注意：`public/` 由 Hexo 自动生成，不需要提交到仓库。但 `hexo deploy` 会自动将 `public/` 推送到 GitHub Pages 的 `main` 分支。

---

## 四、生成并部署博客

### 4.1 清理旧文件

```bash
npx hexo clean
```

### 4.2 生成静态文件

```bash
npx hexo generate
```

### 4.3 部署到 GitHub Pages

```bash
npx hexo deploy
```

> 首次部署时会要求输入 GitHub 用户名和密码（或 Token）。

---

## 五、配置 GitHub Pages

### 5.1 进入仓库设置

1. 打开 `https://github.com/Dhwass01/Dhwass01.github.io`
2. 点击 **Settings** → **Pages**

### 5.2 设置发布源

- **Source**: Deploy from a branch
- **Branch**: `main`，文件夹选择 `/ (root)`
- 点击 **Save**

### 5.3 等待部署

- GitHub Actions 会自动构建和部署
- 通常 1-2 分钟后即可访问

---

## 六、访问博客

- 打开浏览器访问：**https://dhwass01.github.io**
- 如果页面正常显示，说明搭建成功！

## 九、常见问题

### Q1: 部署后页面 404？

- 检查 GitHub Pages 设置中 Branch 是否选择了 `main`
- 确认仓库名为 `<用户名>.github.io`

### Q2: 样式丢失？

- 检查 `_config.yml` 中 `url` 和 `root` 是否正确
- 应为 `url: https://dhwass01.github.io`，`root: /`

### Q3: 部署时认证失败？

- 使用 GitHub Personal Access Token 代替密码
- 在 GitHub → Settings → Developer Settings → Personal Access Tokens 生成

### Q4: 如何回滚？

- Git 记录了所有提交历史，可以使用 `git revert` 回滚
