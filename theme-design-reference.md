## 1. Cactus — 极简主义终端风格

| 属性        | 详情                                                                                |
| ----------- | ----------------------------------------------------------------------------------- |
| **GitHub**  | [probberechts/hexo-theme-cactus](https://github.com/probberechts/hexo-theme-cactus) |
| **Stars**   | ⭐ 3.5k+                                                                            |
| **License** | MIT                                                                                 |
| **Demo**    | https://probberechts.github.io/hexo-theme-cactus/                                   |

### 1.1 设计哲学

Cactus 走的是**极致的极简路线**，灵感来源于终端/命令行界面。整个页面几乎没有多余的装饰元素，信息密度高，阅读体验直接而专注。它追求的是"内容即界面"——去掉一切花哨的视觉元素，只留下文字本身的力量。

### 1.2 布局结构

```
┌──────────────────────────────────────────────┐
│  [左侧栏 ~30%]        │  [右侧主内容 ~70%]    │
│                        │                      │
│  ● 站点标题 (大号)     │  自我介绍段落          │
│  ● 导航菜单            │  社交链接              │
│    - Home              │                      │
│    - About             │  ── Writing ──────    │
│    - Writing           │  14 Nov 2016 标题     │
│    - Projects          │  01 Nov 2016 标题     │
│                        │  26 Dec 2013 标题     │
│                        │                      │
│                        │  ── Projects ─────    │
│                        │  项目名: 描述          │
│                        │  项目名: 描述          │
├──────────────────────────────────────────────┤
│  页脚: Copyright + 导航重复                    │
└──────────────────────────────────────────────┘
```

**核心布局特征：**

- **左右分栏**：左侧固定侧边栏（~30% 宽度）承载站点信息和导航；右侧主区域展示内容
- **首页非传统**：不是文章列表流，而是个人简介 + 精选文章/项目展示
- **文章列表**：简洁的时间轴形式，仅显示日期 + 标题，无缩略图、无摘要
- **页脚**：版权信息 + 导航菜单的重复（方便底部跳转）

### 1.3 配色方案

Cactus 提供 **4 种内置配色方案**：

| 方案名    | 背景色           | 文字色      | 适用场景           |
| --------- | ---------------- | ----------- | ------------------ |
| **White** | `#fff` 纯白      | `#111` 近黑 | 干净明亮的阅读体验 |
| **Dark**  | `#1a1a1a` 深灰黑 | `#aaa` 浅灰 | 夜间/科技感        |
| **Light** | `#f5f5f5` 浅灰白 | `#333` 深灰 | 柔和不刺眼         |
| **Dark2** | `#111` 纯黑      | `#aaa` 浅灰 | 极致暗黑风格       |

**色彩特点：**

- 黑白灰为主基调，几乎不使用彩色
- 链接和强调元素使用微妙的对比色
- 整体色调克制、沉稳，有"黑客/程序员"的审美趣味

### 1.4 排版与字体

- **字体**：等宽字体（Monospace）为首选，呼应终端风格
- **字号层次**：标题较大但不过分夸张，正文适中
- **行间距**：宽松，保证长文阅读舒适度
- **段落间距**：明显，段落之间有清晰的呼吸感

### 1.5 核心功能特性

- ✅ 本地搜索（内置）
- ✅ RSS 订阅
- ✅ 多语言支持（i18n）
- ✅ 响应式设计
- ✅ 社交媒体图标集成
- ✅ 项目展示模块
- ✅ 关于页面
- ✅ MathJax 数学公式支持
- ✅ Google Analytics / Disqus 评论

### 1.6 适用场景

- 个人技术博客 / 开发者主页
- 追求极简、专注内容的写作者
- 喜欢暗色系 / 终端美学的用户
- 不需要复杂视觉装饰的知识分享型网站

## 2. Clean Blog — 大气文艺杂志风格

| 属性         | 详情                                                                                     |
| ------------ | ---------------------------------------------------------------------------------------- |
| **GitHub**   | [klugjo/hexo-theme-clean-blog](https://github.com/klugjo/hexo-theme-clean-blog)          |
| **Stars**    | ⭐ 404                                                                                   |
| **License**  | MIT                                                                                      |
| **Demo**     | https://www.codeblocq.com/assets/projects/hexo-theme-clean-blog/                         |
| **原始模板** | [Start Bootstrap - Clean Blog](http://startbootstrap.com/template-overviews/clean-blog/) |

### 2.1 设计哲学

Clean Blog 的设计理念是**"干净的大气感"**。它借鉴了杂志和出版物的排版风格，用大面积的留白和精致的排版营造出一种"文艺但不矫情"的气质。全屏封面图 + 简约导航的组合让每个页面都有"打开一本杂志"的仪式感。

### 2.2 布局结构

```
┌──────────────────────────────────────────────┐
│  [顶部导航栏 - 固定/半透明]                     │
│  左: 站点标题    右: Home | Archives | Tags    │
├──────────────────────────────────────────────┤
│                                              │
│     ██████████████████████████████           │
│     █                            █           │
│     █     全屏封面大图区域        █           │
│     █                            █           │
│     █   「 Clean Blog 」          █           │
│     █   ─────────────            █           │
│     █   副标题/描述文字           █           │
│     █                            █           │
│     ██████████████████████████████           │
│                                              │
├──────────────────────────────────────────────┤
│  [文章列表区域 - 居中单栏]                      │
│                                              │
│      Welcome to Clean Blog                    │
│      Posted by xxx on March 14th 2016         │
│      ─────────────────────────────            │
│                                              │
│      Images                                   │
│      December 26th 2013                       │
│      ─────────────────────────────            │
│                                              │
│      Untitled                                  │
│      December 25th 2013                       │
│      ─────────────────────────────            │
│                                              │
│              ← Older Posts →                  │
│                                              │
├──────────────────────────────────────────────┤
│  [页脚]                                       │
│  社交图标 (GitHub等)                            │
│  © 2017 John Doe                              │
│  Original Theme: Clean Blog                   │
└──────────────────────────────────────────────┘
```

**核心布局特征：**

- **全屏封面 Hero**：首页顶部是全宽的封面图 + 大标题 + 副标题，视觉冲击力强
- **居中单栏**：文章列表和正文都是居中的单栏布局，阅读体验极佳
- **文章列表**：标题 + 作者/日期信息 + 分割线，层次分明
- **分页**：底部 "Older Posts / Newer Posts" 经典翻页
- **导航栏**：顶部固定，半透明效果，滚动时可见

### 2.3 配色方案

**默认配色：**

- **主背景**：白色 `#fff`
- **文字**：深灰 `#404040`（比纯黑柔和，阅读更舒适）
- **封面区域**：深色调叠层 + 背景图
- **导航栏**：深色半透明 `rgba(0,0,0,0.7)` 或白色
- **链接/强调**：蓝绿色调（Bootstrap 默认色系）
- **分割线**：浅灰 `#eee`

**色彩特点：**

- 白色为主基调，大面积留白
- 封面图提供视觉色彩，正文区域保持素净
- 对比度适中，不追求强烈的黑白对比

### 2.4 排版与字体

- **标题字体**：Serif（衬线体），如 Georgia、Playfair Display，有文学气质
- **正文字体**：Sans-serif（无衬线体），如 Lora、Open Sans，清晰易读
- **字号层次**：封面标题超大（~50px），文章标题中等，正文 16-18px
- **行高**：1.6-1.8，非常适合长文阅读
- **段落间距**：宽松，配合分割线形成节奏感

### 2.5 核心功能特性

- ✅ Bootstrap 框架驱动（响应式完善）
- ✅ 全屏封面图支持（每篇文章可单独配置）
- ✅ Disqus 评论系统集成
- ✅ Google Analytics 统计
- ✅ 标签 (Tags) 和分类 (Categories) 页面
- ✅ 归档 (Archives) 页面
- ✅ 文章内图片展示
- ✅ 视频嵌入支持
- ✅ Gallery 画廊文章
- ✅ 社交媒体链接（页脚）
- ✅ 分页导航

### 2.6 适用场景

- 个人博客 / 写作平台
- 摄影师 / 设计师的作品配文
- 品牌故事 / 创始人信
- 需要"杂志般阅读体验"的内容型网站
- 偏好文艺、干净、大气风格的用户

## 3. Edinburgh — 网格画廊风格

| 属性        | 详情                                                      |
| ----------- | --------------------------------------------------------- |
| **GitHub**  | [A-limon/Edinburgh](https://github.com/A-limon/Edinburgh) |
| **Stars**   | ⭐ 380                                                    |
| **License** | MIT                                                       |
| **Demo**    | https://alimon.github.io/Edinburgh/                       |

### 3.1 设计哲学

Edinburgh 的核心理念是**"用图片讲故事"**。它打破了传统博客"文字堆叠"的模式，采用类似 Dribbble / Behance / Pinterest 的网格卡片布局，让每篇内容都以"视觉作品"的形式呈现。非常适合视觉创作者——摄影师、设计师、旅行者。

### 3.2 布局结构

```
┌──────────────────────────────────────────────┐
│  [顶部导航栏]                                  │
│  左: Edinburgh    右: About | Contact | 社交   │
├──────────────────────────────────────────────┤
│                                              │
│  ┌──────────┐  ┌──────────┐                  │
│  │          │  │          │                  │
│  │  封面图1  │  │  封面图2  │                  │
│  │          │  │          │                  │
│  │ 标题1    │  │ 标题2    │                  │
│  └──────────┘  └──────────┘                  │
│  ┌──────────┐  ┌──────────┐                  │
│  │          │  │          │                  │
│  │  封面图3  │  │  封面图4  │                  │
│  │          │  │          │                  │
│  │ 标题3    │  │ 标题4    │                  │
│  └──────────┘  └──────────┘                  │
│                                              │
├──────────────────────────────────────────────┤
│  [文章详情页]                                  │
│                                              │
│  ┌──────────────────────────────────┐        │
│  │        全宽大封面图               │        │
│  │                                  │        │
│  └──────────────────────────────────┘        │
│         文章标题 (居中大字)                    │
│         正文内容                              │
│                                              │
└──────────────────────────────────────────────┘
```

**核心布局特征：**

- **2×N 网格**：首页是 2 列等宽卡片网格，每张卡片 = 一个带封面图的项目/文章
- **卡片设计**：纯图片 + 标题叠加在图片底部，无多余信息
- **文章详情**：全宽大封面图 + 居中标题 + 下方正文，视觉冲击力极强
- **导航简洁**：顶部左侧 Logo，右侧少量链接 + 社交图标
- **无侧边栏**：内容区域全宽，最大化利用空间展示图片

### 3.3 配色方案

**默认配色：**

- **主背景**：纯白 `#fff`
- **卡片文字**：白色（叠加在图片上，有阴影/渐变遮罩确保可读性）
- **导航栏**：白色背景，深色文字
- **正文**：深灰 `#333`
- **强调色**：根据封面图自适应，无固定主题色

**色彩特点：**

- 极度"图片驱动"——页面的色彩完全取决于封面图
- 白色背景让图片成为绝对主角
- 干净、现代、有设计感

### 3.4 排版与字体

- **导航/标题**：Sans-serif（无衬线体），干净现代
- **正文**：Sans-serif 或 Serif，视具体配置
- **卡片标题**：白色，中等字号，叠加在图片底部
- **文章标题**：居中、大字号，视觉焦点
- **正文排版**：居中对齐或左对齐，行高舒适

### 3.5 核心功能特性

- ✅ 网格卡片布局（作品展示）
- ✅ 封面图自动裁剪/缩放
- ✅ 文章详情页全宽大图
- ✅ 响应式设计
- ✅ 社交媒体集成（Instagram、LinkedIn、Facebook 等）
- ✅ About 页面
- ✅ Contact 联系页
- ✅ 简洁导航

### 3.6 适用场景

- 摄影师作品集 / 摄影博客
- 设计师 Portfolio
- 旅行博客（图片为主）
- 美食 / 生活方式博客
- 任何以**视觉内容为核心**的网站
- 希望"作品展示优先，文字介绍为辅"的创作者

## 5. 设计要素提炼与复用指南

### 5.1 可复用的布局模式

#### 模式 A：左右分栏（来自 Cactus）

```
适用场景：个人主页、技术文档、知识库
核心元素：
- 固定左侧栏（Logo + 导航 + 社交链接）
- 右侧可滚动内容区
- 移动端自动堆叠为上下布局
优点：导航始终可见，内容区空间独立
```

#### 模式 B：全屏 Hero + 单栏（来自 Clean Blog）

```
适用场景：博客、文章站、品牌故事
核心元素：
- 首屏全宽封面图（100vh 或 60-80vh）
- 居中单栏正文（max-width: 700-800px）
- 固定/半透明顶部导航
优点：视觉冲击力强，阅读体验专注
```

#### 模式 C：网格卡片（来自 Edinburgh）

```
适用场景：作品集、摄影站、产品展示
核心元素：
- 2-3 列等宽网格卡片
- 卡片以封面图为主，标题叠加
- 悬停效果（放大/遮罩/标题滑入）
优点：浏览效率高，视觉吸引力强
```

### 5.2 可复用的配色策略

| 策略           | 说明                                              | 来源        |
| -------------- | ------------------------------------------------- | ----------- |
| **纯黑白灰**   | 无彩色方案，极致克制，适合内容为王的场景          | Cactus      |
| **白底深文**   | 白色背景 + `#404040` 文字（非纯黑），减少视觉疲劳 | Clean Blog  |
| **图片驱动色** | 白色中性底色，页面色彩完全由封面图决定            | Edinburgh   |
| **暗色模式**   | `#1a1a1a` 背景 + 浅灰文字，护眼且有科技感         | Cactus Dark |

### 5.3 可复用的排版原则

1. **行高 1.6-1.8**：长文阅读的关键舒适区（三者共同点）
2. **正文宽度 600-800px**：过宽影响阅读追踪，过窄浪费空间（Clean Blog 最佳实践）
3. **标题层次分明**：H1 用于封面大标题（40-50px），H2 用于文章标题（24-30px），H3 用于段落标题（18-20px）
4. **留白即设计**：三者都大量使用留白，不会让页面"填满"
5. **分割线/间距**：用简洁的线条或充足的间距来分隔内容块，而非花哨的装饰

### 5.4 混搭建议

| 组合方案                             | 说明                                                                                |
| ------------------------------------ | ----------------------------------------------------------------------------------- |
| **Cactus 布局 + Clean Blog 配色**    | 左侧栏结构 + 白色温暖配色，适合个人技术博客                                         |
| **Clean Blog Hero + Edinburgh 网格** | 全屏封面作为站点入口，下方用网格展示文章，适合混合型博客                            |
| **Edinburgh 网格 + Cactus 暗色**     | 暗色背景下的图片网格，适合摄影作品集                                                |
| **三者融合**                         | 首页用 Edinburgh 网格展示精选内容，文章页用 Clean Blog 排版，侧边导航用 Cactus 风格 |

## 附录：快速安装参考

```bash
# 安装 Cactus
cd themes
git clone https://github.com/probberechts/hexo-theme-cactus.git cactus

# 安装 Clean Blog
cd themes
git clone https://github.com/klugjo/hexo-theme-clean-blog.git clean-blog

# 安装 Edinburgh
cd themes
git clone https://github.com/A-limon/Edinburgh.git edinburgh

# 修改 _config.yml 中的 theme 字段
# theme: cactus / clean-blog / edinburgh

# 生成并预览
hexo clean && hexo g && hexo s
```
