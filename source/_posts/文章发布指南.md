# 📝 Hexo 博客文章发布指南

> 本文档帮助你快速掌握如何撰写、发布和管理博客文章。

---

## 一、文章放在哪里？

所有文章都存放在这个目录下：

```
d:\BlogFile\source\_posts\
```

每篇文章是一个 `.md`（Markdown）文件，例如：

- `futurism-theme-guide.md`
- `hello-world.md`

---

## 二、创建新文章

### 方式一：手动创建（推荐）

1. 在 `source\_posts\` 目录下新建一个 `.md` 文件
2. 文件名就是文章的 URL 路径，例如：
   - 文件名：`my-first-post.md` → URL：`/2026/05/26/my-first-post/`
   - 建议使用英文、数字和连字符，避免中文文件名

### 方式二：使用 Hexo 命令

```bash
npx hexo new "文章标题"
```

这会在 `source\_posts\` 下自动生成一个带日期前缀的 `.md` 文件。

---

## 三、文章格式（Front Matter）

每篇文章的开头必须有一段 **Front Matter**，用 `---` 包裹：

```markdown
---
title: 文章标题
date: 2026-05-26 15:00:00
updated: 2026-05-26 18:00:00
tags:
  - 标签1
  - 标签2
categories:
  - 分类名
cover: /images/cover.jpg
excerpt: 这是一段摘要，会显示在首页卡片上。
---

这里是文章正文内容...
```

### 字段说明

| 字段         | 必填 | 说明                                      |
| ------------ | ---- | ----------------------------------------- |
| `title`      | ✅   | 文章标题                                  |
| `date`       | ✅   | 发布日期，格式：`YYYY-MM-DD HH:mm:ss`     |
| `updated`    | ❌   | 更新日期（不填则不显示）                  |
| `tags`       | ❌   | 标签列表，支持多个                        |
| `categories` | ❌   | 分类（建议只用一个）                      |
| `cover`      | ❌   | 封面图片路径（不填则显示渐变色占位）      |
| `excerpt`    | ❌   | 自定义摘要（不填则自动截取正文前 120 字） |

## 四、Markdown 写作速查



## 五、插入图片

### 方式一：放在 source/images/ 目录（推荐）

1. 将图片放到 `source\images\` 目录下
2. 在文章中引用：

```markdown
![描述](/images/my-photo.jpg)
```

### 方式二：文章同名文件夹

1. 在 `source\_posts\` 下创建与文章同名的文件夹：

```
source\_posts\
├── my-post.md
└── my-post\
    └── photo.jpg
```

2. 在文章中引用：

```markdown
![描述](my-post/photo.jpg)
```

### 方式三：使用外链

```markdown
![描述](https://i.imgur.com/xxxxx.jpg)
```

## 六、部署架构

在讲发布流程之前，先了解博客的部署架构，这样遇到问题时知道该排查哪一层。

```
┌─────────────────────────────────────────────────────┐
│                    你的电脑                           │
│  source/_posts/xxx.md  ──→  git push                │
└──────────────────────┬──────────────────────────────┘
                       │ push 到 main 分支
                       ▼
┌─────────────────────────────────────────────────────┐
│              GitHub (main 分支)                      │
│  存放源码：Markdown 文件、主题代码、配置文件           │
└──────────────────────┬──────────────────────────────┘
                       │ 自动触发 GitHub Actions
                       ▼
┌─────────────────────────────────────────────────────┐
│           GitHub Actions (自动构建)                   │
│  1. 安装依赖 (npm ci)                               │
│  2. 构建站点 (hexo generate)                        │
│  3. 部署到 gh-pages 分支                             │
└──────────────────────┬──────────────────────────────┘
                       │ 构建产物推送到 gh-pages 分支
                       ▼
┌─────────────────────────────────────────────────────┐
│           GitHub Pages (gh-pages 分支)               │
│  存放构建好的 HTML/CSS/JS 静态文件                    │
└──────────────────────┬──────────────────────────────┘
                       │ 提供网页服务
                       ▼
┌─────────────────────────────────────────────────────┐
│         https://dhwass01.github.io                  │
│                  博客上线！                           │
└─────────────────────────────────────────────────────┘
```

### 关键文件

| 文件 | 作用 |
|------|------|
| `_config.yml` | 站点配置（标题、URL、主题等） |
| `source/_posts/*.md` | 文章源文件 |
| `themes/futurism/` | Futurism 主题文件 |
| `.github/workflows/deploy.yml` | GitHub Actions 自动构建部署脚本 |
| `.gitignore` | 控制哪些文件不上传到 GitHub |

> ⚠️ **重要**：`public/` 目录（构建产物）已在 `.gitignore` 中排除，不会上传到 GitHub。构建由 GitHub Actions 在云端自动完成。

## 七、上传文章（完整流程）

### 第 1 步：写好文章

在 `source\_posts\` 下创建 `.md` 文件，写好内容。确保文件开头有正确的 Front Matter：

```markdown
---
title: 我的新文章
date: 2026-05-26 15:00:00
tags:
  - 标签1
  - 标签2
categories:
  - 分类
excerpt: 一句话摘要。
---

文章正文...
```

### 第 2 步：本地预览

```bash
# 进入博客目录
cd d:\BlogFile

# 启动本地服务器
npx hexo server
```

打开浏览器访问 `http://localhost:4000` 预览效果。

> 💡 **提示**：如果遇到 PowerShell 安全限制，使用 `cmd /c "npx hexo server"`
>
> 💡 **提示**：修改文章后刷新页面即可看到更新，无需重启服务器

### 第 3 步：提交并推送

确认预览效果满意后：

```bash
cd d:\BlogFile

# 1. 暂存所有更改
git add -A

# 2. 提交（引号内写本次更新说明）
git commit -m "feat: 新增文章 - 我的新文章"

# 3. 临时禁用 GCM（解决推送认证问题，详见下方踩坑记录）
git config --global credential.helper ""

# 4. 推送
git push -u origin main

# 5. 恢复 GCM
git config --global credential.helper manager
```

### 第 4 步：等待自动部署

推送成功后，GitHub Actions 会自动执行以下步骤：

1. 检出 main 分支代码
2. 安装 Node.js 20 + 项目依赖
3. 运行 `hexo generate` 生成静态文件
4. 将 `public/` 目录部署到 `gh-pages` 分支

**整个过程通常需要 1-2 分钟**，之后即可在 https://dhwass01.github.io 看到新文章。

### 第 5 步：验证上线

访问 https://dhwass01.github.io ，确认新文章出现在首页。

如需检查部署状态，访问：`https://github.com/Dhwass01/Dhwass01.github.io/actions`

## 八、下架文章（完整流程）

### 场景一：暂时下架（推荐）

如果不想删除文章源文件，只是不想让它出现在博客上：

1. **将文件移到草稿目录**：

```bash
# 创建草稿目录（如果不存在）
mkdir -p source/_drafts

# 移动文件
move source\_posts\文章名.md source\_drafts\文章名.md
```

2. **提交推送**：

```bash
git add -A
git commit -m "draft: 暂时下架 - 文章名"
git config --global credential.helper ""
git push -u origin main
git config --global credential.helper manager
```

3. **恢复上架**：将文件从 `_drafts` 移回 `_posts`，提交推送即可。

### 场景二：永久删除

1. **删除文章文件**：

```bash
del source\_posts\文章名.md
```

2. **如有同名图片目录，一并删除**：

```bash
rmdir /s /q source\_posts\文章名
```

3. **提交推送**：

```bash
git add -A
git commit -m "remove: 删除文章 - 文章名"
git config --global credential.helper ""
git push -u origin main
git config --global credential.helper manager
```

### 场景三：批量操作

如果要同时下架多篇文章：

```bash
# 1. 逐个移走或删除文件
move source\_posts\文章1.md source\_drafts\文章1.md
move source\_posts\文章2.md source\_drafts\文章2.md
del source\_posts\文章3.md

# 2. 一次性提交
git add -A
git commit -m "refactor: 批量整理文章"
git config --global credential.helper ""
git push -u origin main
git config --global credential.helper manager
```

> 💡 **提示**：GitHub Actions 每次都会重新构建整个站点，所以下架后文章页面会在 1-2 分钟内消失。但注意，如果搜索引擎已经收录了该页面，可能需要一段时间才会从搜索结果中移除。

## 十、注意事项

### 文件命名

- ✅ 推荐：`my-first-post.md`、`hexo-guide.md`
- ❌ 避免：`我的文章.md`（中文文件名可能导致 URL 编码问题）

### 日期格式

- ✅ 正确：`2026-05-26 15:00:00`
- ❌ 错误：`2026/5/26`、`26-05-2026`

### 标签和分类

- 标签可以有多个
- 分类建议只用一个（Hexo 的分类有层级关系）
- 如果文章没有分类，可以不写 `categories` 字段

### 封面图片

- 建议尺寸：1200×630 像素（适合社交分享）
- 支持格式：jpg、png、webp、gif
- 如果不填 `cover` 字段，首页卡片会显示渐变色占位背景

### Git Commit 规范

建议使用以下前缀，让提交历史更清晰：

| 前缀 | 用途 | 示例 |
|------|------|------|
| `feat:` | 新增文章/功能 | `feat: 新增文章 - CSS 技巧` |
| `fix:` | 修复问题 | `fix: 修复文章中的拼写错误` |
| `draft:` | 暂时下架文章 | `draft: 暂时下架 - 旧文章` |
| `remove:` | 永久删除文章 | `remove: 删除文章 - 过期内容` |
| `refactor:` | 批量整理 | `refactor: 批量更新标签` |
| `ci:` | 部署/构建相关 | `ci: 更新 GitHub Actions 配置` |
| `style:` | 样式调整 | `style: 调整首页卡片间距` |

## 十二、踩坑记录：首次部署遇到的问题

以下是搭建博客和首次部署 GitHub Pages 时遇到的所有问题及解决方案，供日后参考。

### 问题 1：GitHub Pages 显示旧主题（Landscape）

**现象**：代码已经推送到 GitHub，但打开 https://dhwass01.github.io 显示的是 Hexo 默认的 Landscape 主题，而不是 Futurism 主题。

**原因**：`public/` 目录在 `.gitignore` 中被排除，构建好的 HTML 文件根本没有上传到 GitHub。GitHub Pages 无法获取到正确的静态文件。

**解决方案**：配置 GitHub Actions 自动构建部署。创建 `.github/workflows/deploy.yml`，让 GitHub 在云端自动执行 `hexo generate` 并将产物部署到 `gh-pages` 分支。

```yaml
# .github/workflows/deploy.yml
name: Deploy Hexo to GitHub Pages

on:
  push:
    branches:
      - main

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx hexo generate
      - uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
          publish_branch: gh-pages
```

同时需要在 GitHub 仓库的 Settings → Pages 中，将 Source 从 `main` 分支改为 `gh-pages` 分支。

---

### 问题 2：GitHub Actions 工作流文件推送失败

**现象**：`git push` 时报错：

```
refusing to allow a Personal Access Token to create or update workflow
`.github/workflows/deploy.yml` without `workflow` scope
```

**原因**：GitHub 对 Personal Access Token (PAT) 的权限管理非常严格。创建或修改 `.github/workflows/` 下的文件需要 `workflow` 权限，而普通的 PAT 默认没有这个权限。

**解决方案**：

1. 打开 GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
2. 编辑你正在使用的 Token
3. 在 Repository permissions 中添加：
   - `Contents` → Read and write
   - **`Workflows` → Read and write**（关键！）
4. 保存后重新推送即可

> 💡 **提示**：如果是 classic token，需要勾选 `repo` 和 `workflow` 两个 scope。

---

### 问题 3：GCM 拦截 Git 推送认证

**现象**：执行 `git push` 后，系统弹出浏览器要求登录 GitHub，而不是使用已配置的 Token。

**原因**：Windows 上安装了 Git Credential Manager (GCM)，它会拦截所有 Git 认证请求，尝试弹出 OAuth 浏览器登录，即使 Token 已经嵌入在远程 URL 中。

**解决方案**：推送前临时禁用 GCM，推送后恢复：

```bash
# 禁用 GCM
git config --global credential.helper ""

# 推送
git push -u origin main

# 恢复 GCM
git config --global credential.helper manager
```

> 💡 **提示**：每次推送都需要执行这三步。可以写成批处理脚本简化操作。

---

### 问题 4：Front Matter 缺少导致构建失败

**现象**：本地执行 `hexo generate` 时报 YAML 解析错误：

```
YAMLException: unexpected end of the stream
```

**原因**：某篇文章缺少了 `---` 包裹的 Front Matter 块。Hexo 要求每篇文章必须以 `---` 开头和结尾的 YAML 块作为元数据。

**解决方案**：确保每篇文章的开头都有完整的 Front Matter：

```markdown
---
title: 文章标题
date: 2026-05-26 15:00:00
---

正文内容...
```

> 💡 **提示**：即使不需要标签和分类，`title` 和 `date` 也不能省略，`---` 分隔符更不能少。

---

### 问题 5：PowerShell 中 `&&` 不可用

**现象**：在 PowerShell 中执行 `cd d:\BlogFile && git status` 报错 `CommandNotFoundException`。

**原因**：PowerShell 不支持 `&&` 语法（这是 bash/cmd 的语法）。PowerShell 中的命令分隔符是 `;`。

**解决方案**：

```powershell
# ❌ 错误
cd d:\BlogFile && git status

# ✅ 正确（PowerShell）
cd d:\BlogFile; git status
```

---

### 问题 6：PowerShell 中 `cmd /c` 报错

**现象**：执行 `cmd /c "npx hexo server"` 报错 `CommandNotFoundException`。

**原因**：某些 PowerShell 配置下，`cmd` 命令被别名覆盖或路径解析异常。

**解决方案**：直接使用 `npx` 命令，或通过 `Start-Process` 调用：

```powershell
# 直接用 npx
npx hexo server

# 或者用 Start-Process
Start-Process -FilePath "cmd" -ArgumentList "/c npx hexo server" -Wait -NoNewWindow
```

---

### 问题 7：字数统计插件显示的是"词"不是"字"

**现象**：安装了 `hexo-wordcount` 插件，但显示的数字比预期小很多（例如一篇长文只显示 1.9k）。

**原因**：`hexo-wordcount` 的 `wordcount()` 方法统计的是英文意义上的"词"（word），对中文来说是以标点和空格分隔的单元，不等于字符数。例如 5500+ 个中文字符可能只统计为 1.9k 个"词"。

**解决方案**：了解这个特性即可，1.9k words 大约对应 8 分钟的阅读时长，这个估算基本合理。如果需要精确的中文字符统计，可以使用 `symbolcount()` 方法。

---

### 问题 8：Geist 字体 CDN 的 MIME 类型问题

**现象**：本地浏览器控制台报错，Geist 字体 CSS 文件被 ORB（Origin Read Blocking）拦截。

**原因**：jsdelivr CDN 对某些 CSS 文件返回了不正确的 MIME 类型（`text/plain` 而不是 `text/css`），导致浏览器拒绝加载。

**解决方案**：将字体 CDN 从 jsdelivr 切换到 unpkg：

```yaml
# themes/futurism/_config.yml
font:
  cdn: "https://unpkg.com/geist@1.4.1/dist/fonts/geist-sans/style.css"
```

> 💡 **提示**：这个 ORB 拦截主要发生在本地预览，线上部署后通常不受影响。

## 十三、常用命令速查

```bash
# ──── 本地开发 ────
npx hexo clean                    # 清理缓存
npx hexo generate                 # 生成静态文件
npx hexo server                   # 启动本地服务器 (localhost:4000)
npx hexo new "文章标题"           # 新建文章

# ──── Git 操作 ────
git status                        # 查看文件变更
git add -A                        # 暂存所有更改
git commit -m "说明"              # 提交
git config --global credential.helper ""   # 禁用 GCM
git push -u origin main           # 推送
```
