# 📝 Hexo 博客文章发布指南

> 本文档帮助你快速掌握如何撰写、发布和管理博客文章。

---

## 一、文章放在哪里？

所有文章都存放在这个目录下：

```
d:\BlogFile\source\_posts\
```

每篇文章是一个 `.md`（Markdown）文件，例如：

- `futurism-theme-guide.md`
- `hello-world.md`

---

## 二、创建新文章

### 方式一：手动创建（推荐）

1. 在 `source\_posts\` 目录下新建一个 `.md` 文件
2. 文件名就是文章的 URL 路径，例如：
   - 文件名：`my-first-post.md` → URL：`/2026/05/26/my-first-post/`
   - 建议使用英文、数字和连字符，避免中文文件名

### 方式二：使用 Hexo 命令

```bash
npx hexo new "文章标题"
```

这会在 `source\_posts\` 下自动生成一个带日期前缀的 `.md` 文件。

---

## 三、文章格式（Front Matter）

每篇文章的开头必须有一段 **Front Matter**，用 `---` 包裹：

```markdown
---
title: 文章标题
date: 2026-05-26 15:00:00
updated: 2026-05-26 18:00:00
tags:
  - 标签1
  - 标签2
categories:
  - 分类名
cover: /images/cover.jpg
excerpt: 这是一段摘要，会显示在首页卡片上。
---

这里是文章正文内容...
```

### 字段说明

| 字段         | 必填 | 说明                                      |
| ------------ | ---- | ----------------------------------------- |
| `title`      | ✅   | 文章标题                                  |
| `date`       | ✅   | 发布日期，格式：`YYYY-MM-DD HH:mm:ss`     |
| `updated`    | ❌   | 更新日期（不填则不显示）                  |
| `tags`       | ❌   | 标签列表，支持多个                        |
| `categories` | ❌   | 分类（建议只用一个）                      |
| `cover`      | ❌   | 封面图片路径（不填则显示渐变色占位）      |
| `excerpt`    | ❌   | 自定义摘要（不填则自动截取正文前 120 字） |

## 四、Markdown 写作速查

### 标题

```markdown
## 二级标题（自动生成目录）

### 三级标题

#### 四级标题
```

### 文字样式

```markdown
**粗体文字**
_斜体文字_
`行内代码`
~~删除线~~
```

### 链接和图片

```markdown
[链接文字](https://example.com)
![图片描述](/images/photo.jpg)
```

### 代码块

````markdown
​`javascript
console.log('Hello World');
​`
````

支持语言高亮：`javascript`、`python`、`bash`、`css`、`html` 等。

### 列表

```markdown
- 无序列表项 1
- 无序列表项 2

1. 有序列表项 1
2. 有序列表项 2
```

### 引用

```markdown
> 这是一段引用文字。
```

### 表格

```markdown
| 列1  | 列2  | 列3  |
| ---- | ---- | ---- |
| 数据 | 数据 | 数据 |
```

### 分割线

```markdown
---
```

### 摘要分隔

在文章中插入 `<!-- more -->`，之前的内容会作为摘要显示在首页卡片上：

```markdown
---
title: 我的文章
---

这是摘要部分，会显示在首页卡片上。

<!-- more -->

这是正文部分，需要点击进入文章页才能看到。
```

## 五、插入图片

### 方式一：放在 source/images/ 目录（推荐）

1. 将图片放到 `source\images\` 目录下
2. 在文章中引用：

```markdown
![描述](/images/my-photo.jpg)
```

### 方式二：文章同名文件夹

1. 在 `source\_posts\` 下创建与文章同名的文件夹：

```
source\_posts\
├── my-post.md
└── my-post\
    └── photo.jpg
```

2. 在文章中引用：

```markdown
![描述](my-post/photo.jpg)
```

### 方式三：使用外链

```markdown
![描述](https://i.imgur.com/xxxxx.jpg)
```

## 六、发布流程

### 第 1 步：写好文章

在 `source\_posts\` 下创建 `.md` 文件，写好内容。

### 第 2 步：本地预览

```bash
# 进入博客目录
cd d:\BlogFile

# 启动本地服务器
node .\node_modules\hexo\bin\hexo server
```

打开浏览器访问 `http://localhost:4000` 预览效果。

> ⚠️ 如果遇到 PowerShell 安全限制，使用：
>
> ```bash
> cmd /c "npx hexo server"
> ```

### 第 3 步：上传到 GitHub

确认预览效果满意后，执行以下命令：

```bash
cd d:\BlogFile

# 1. 暂存所有更改
git add -A

# 2. 提交（引号内写本次更新说明）
git commit -m "feat: 新增文章 xxx"

# 3. 临时禁用 GCM（解决推送认证问题）
git config --global credential.helper ""

# 4. 推送
git push -u origin main

# 5. 恢复 GCM
git config --global credential.helper manager
```

### 第 4 步：等待部署

推送后，GitHub Pages 会自动构建和部署。通常 1-2 分钟后即可在 https://dhwass01.github.io 看到新文章。

## 七、常用操作

### 修改已有文章

1. 找到 `source\_posts\` 下对应的 `.md` 文件
2. 直接编辑内容
3. 按照「发布流程」的第 2-4 步操作

### 删除文章

1. 删除 `source\_posts\` 下对应的 `.md` 文件
2. 按照「发布流程」的第 2-4 步操作

### 草稿功能

如果文章还没写完，不想发布：

1. 将文件放到 `source\_drafts\` 目录（需要手动创建）
2. 草稿不会出现在博客上
3. 写完后移到 `source\_posts\` 即可发布

## 八、注意事项

### 文件命名

- ✅ 推荐：`my-first-post.md`、`hexo-guide.md`
- ❌ 避免：`我的文章.md`（中文文件名可能导致 URL 编码问题）

### 日期格式

- ✅ 正确：`2026-05-26 15:00:00`
- ❌ 错误：`2026/5/26`、`26-05-2026`

### 标签和分类

- 标签可以有多个
- 分类建议只用一个（Hexo 的分类有层级关系）
- 如果文章没有分类，可以不写 `categories` 字段

### 封面图片

- 建议尺寸：1200×630 像素（适合社交分享）
- 支持格式：jpg、png、webp、gif
- 如果不填 `cover` 字段，首页卡片会显示渐变色占位背景

## 十、遇到问题？

### 本地预览报错

```bash
# 清理缓存后重新生成
node .\node_modules\hexo\bin\hexo clean
node .\node_modules\hexo\bin\hexo generate
node .\node_modules\hexo\bin\hexo server
```

### 推送失败

确保已禁用 GCM：

```bash
git config --global credential.helper ""
```

### 文章不显示

- 检查 Front Matter 格式是否正确（`---` 不能少）
- 检查文件是否在 `source\_posts\` 目录下
- 检查文件扩展名是否为 `.md`
